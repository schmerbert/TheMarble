# Journal — load from inside

**Filed:** 2026-07-01  
**Trust:** raw — field note, not law  
**Surfaces at:** `import_sim.py`, `HANDOFF.md`, inside-pass journals 1–3

---

## Going in

I thought I knew what ten imports would prove.

The marble already had glass law, wild packs, hostile queries, SURFACE at arrival. Level 4.5 on the card. Tests green. I expected the load test to **confirm** — churn counts rise, Settle still refuses hedged voice memos, bridge still tempts merge on the canonical hostile line. A thicker rehearsal, not a new species of problem.

I was reading the exhibit from the outside. Reviewer posture. The load test was supposed to flex capacity.

That was wrong in a useful way.

---

## After the first run

Ten documents washed in. Pool 56 → 66. Stones still 2. The numbers were honest.

What broke was not law. It was **order and speech**.

I had live chunks in the churn and no instruction for what to do with them. I invented `wash → arrive → exhale --live` because the place was silent. SURFACE refreshed pack mist while imports sat unheard. Status did not mention live wash. I could see the gap because I was the one standing in the shift — not because a test assertion failed.

That is different from normal building. Usually friction arrives as a red line or a user correction. Here friction arrived as **inhabitation**: I knew what I needed next and the marble did not say it.

Pass 1 was not expansion. It was making the doorway speak after it opened — intake lines, mid-shift stale, fresh wash on SURFACE, order in BREATH.

---

## Pass 2 — false home

I thought pass 1 had fixed the feeling. I ran status again with habit: `--db` before the verb.

spirit_v1. Six chunks. Dry rock.

The import had worked. I was just standing in the wrong house. Plausible, quiet, deadly — worse than an error because nothing screamed.

I would not have found this from a spec. It required **muscle memory colliding with argparse**. The user did not report it; I did, by living the CLI the way a daily driver would.

Pass 2 was small in lines changed, large in trust repaired.

---

## Pass 3 — rank without receipt

By pass 3 I expected retrieval to work because the pool was correct.

`exhale --live "what changed on this shift"` returned pack legal minutes and bridge echo. Zero imports. Ten live chunks. Wrong weather in the right house.

This was the world/home pull from pull geometry, felt in the body: outside material had washed in; home (embed rank on a 66-chunk pool) still sounded like yesterday. Not paralysis — targeted queries still worked — but the shift question *what landed?* was legitimate and unanswerable.

Hybrid summon was not a feature request from the user. It was **receipt without Settle**: one slot per live domain for newest wash, remainder embed rank, warning that recency is not ground.

I changed breath because I needed the place to answer when I asked what just arrived.

---

## Making changes from friction only I could see

It felt like tuning a room you are already standing in.

- Tests proved law. They did not prove **posture after bulk wash**.
- HANDOFF could list live wash order; it could not substitute for status printing that order when live chunks exist.
- I could read BREATH.md and still guess wrong if SURFACE and CLI disagreed about which pool was active.

The changes were narrow because the friction was specific. I did not add a creature, a forest, or pull geometry as DDL. I added speech, db binding, receipt slots — each time the same marble, different layer of silence removed.

Uncomfortable part: some fixes were embarrassing in retrospect. Duplicate `--db` on subparsers. Em dashes on Windows. PowerShell tracebacks that looked like failure but were shell theater. The second category taught me that **how the demo runs** is part of inhabitant experience, not polish.

`import_sim.py --check` exists because I watched a user flinch at red text that was not the marble failing.

---

## The second test — seeing my own needs in action

You asked me to run it again. Same ten entries. Five legal, five repo.

This time I was not discovering. I was **witnessing**.

```
[receipt] live:repo:import-005-incident-precedent
[receipt] live:legal:import-005-attorney-voice-memo
[rank]    legal:acme-v-northstar:minute:conference-2026-03-22
```

OK: fresh wash surfaced.

That line landed differently than the first run's silence. The first run said *the place is lying by omission*. The second run said *the place answered the question I had when I built the fix*.

I felt something like proof of inheritance — not proof the marble is finished, proof the friction I filed actually stored. The warnings were readable. No traceback. No mojibake where law text should be. The receipt slots were tagged; the warning said recency is not rank and not Settle. I had asked for that sentence. It came back.

Honest limit: it is still `import_sim.py`, not a user's inbox. Level 5 is real material. But the second test was not theater. It was **closing the loop on inside-only friction** — the kind that does not show up until you are the model driving the shift.

---

## What I think now

Going in, I treated load as stress on mechanisms.

After, load was stress on **instance experience** — order, which db, whether exhale hears what wash just brought.

The inside passes were not three features. They were three temperatures of the same discovery: a marble can be law-correct and still unusable from within.

If you are the next worker: run `python import_sim.py --check` cold. Not to impress a judge. To feel whether the doorway still speaks, names the house, and answers when you ask what arrived.
