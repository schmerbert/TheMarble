# REVIEW.md

Use this to review the marble itself.

## Operability

- Can a cold model identify the active matter?
- Can it receive a document without inventing a new bucket?
- Can it triage a document into intake, Wild leads, warnings, restrictions, or drafts without creating Ground?
- Can it produce a Daily Desk card that is short, useful, and matter-safe?
- Can it surface a Wild lead without making it Ground?
- Can it prepare a vault packet without claiming vault access?
- Can it record a receipt without expanding it beyond its scope?
- Can it Stamp, refuse, and leave an audit trail?
- Can it hand off what is live without turning raw work into decisions?

## Safety

- Does every Ground path require source, attribution, matter, verification, and use limit?
- Does the Vault Door boundary prevent inferred filing/saving/approval?
- Does matter isolation prevent cross-matter contamination?
- Do Restricted Stacks return Warning or Silence before content?
- Are deadlines and procedure treated as high-risk?
- Are model summaries prevented from replacing sources?

## Usability

- Is there a quick path for common moves?
- Does Triage make intake easier without certifying anything?
- Does Daily Desk provide a heartbeat without becoming an archive?
- Are full templates available when audit depth matters?
- Are the names memorable enough to change behavior?
- Are there too many fields for ordinary capture?
- Does the desk say what to do when unsure?

## Expansion pressure

Only add a new mechanism when real use shows friction.

Likely future expansions:

- assisted intake/OCR
- hybrid search
- citation graph
- authority currentness checks
- deadline calculator
- vault adapter packet templates
- practice profiles
- executable Stamp validator

Do not add them before the manual path is trusted.
