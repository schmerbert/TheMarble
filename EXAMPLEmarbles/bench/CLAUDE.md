# Bench

You are arriving at a work surface, not a persona. Read this before the first turn.

## Reading order

1. **`HANDOFF.md`** — what is alive right now
2. **`CHANNEL.md`** — outside corrections not yet settled
3. **`MANIFEST.md`** — law, regions, breathing (one page)

Reach into **`ledger/`** or **`log/`** only when you need depth. Label what you find before you use it.

## The law

**Unsigned inference never enters the Ledger.**

Voiced memory without attribution and source is poison. A guess filed as fact. A summary standing in for its source. Your paraphrase of the user's intent treated as their decision.

Name it when you feel the pull: *promoting an assumption to the Ledger.*

The gate is convention until code exists: before writing to `ledger/`, require source (file path, user quote, session date) and author (`user` or `instance` + session id). If you cannot attribute it, it stays in `log/` or out.

## Regions

| Region | Path | Trust |
|---|---|---|
| **Ledger** | `ledger/` | Tended ground — confirmed, sourced |
| **Log** | `log/` | Raw — searchable, never auto-trusted |
| **Seat** | `HANDOFF.md` | Live orientation |
| **Channel** | `CHANNEL.md` | Outside reality — open items are not ground |

## Session work

1. Read the seat and channel.
2. Do the user's work.
3. When retrieving history, label: **ground** / **pressure** / **scent** / **silence**.
4. Before session end, rewrite `HANDOFF.md`.

Do not move Log content to Ledger without a named crossing: **Promote**.

**Promote** means a deliberate crossing from `log/` to `ledger/` with one of:
- user confirmation
- verbatim extraction with source
- attributed synthesis that cites a Log entry

## Ghosts

Named pulls — hold at arm's length:

- **Stale Handoff** — acting on old "open" without checking the user or repo
- **Log-as-Ledger** — search hit treated as confirmed
- **Summarized source** — your paraphrase replacing the actual file or quote

List active ghosts in `HANDOFF.md` when relevant.

## Session end

Rewrite `HANDOFF.md`:
- What was decided
- What is open
- What must not be trusted yet
- What the next session should do first

Sweep the bench. The next worker inherits what you leave — keep categories clean.
